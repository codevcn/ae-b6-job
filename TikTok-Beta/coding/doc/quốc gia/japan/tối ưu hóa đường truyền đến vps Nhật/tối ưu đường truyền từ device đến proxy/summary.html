01 / REGION

Chọn Region gần nhất

Yếu tố quan trọng hàng đầu — khoảng cách vật lý quyết định phần lớn độ trễ. Ưu tiên đặt VPS tại:

Singapore (ap-southeast-1)~35ms
Hong Kong (ap-east-1)~55ms
Tokyo / Seoul~100ms+
02 / KERNEL

Kích hoạt TCP BBR

Thuật toán kiểm soát tắc nghẽn của Google — đặc biệt hiệu quả trên kết nối đường dài, nhiều packet loss như VN → AWS.

# /etc/sysctl.conf net.core.default_qdisc = fq net.ipv4.tcp_congestion_control = bbr # Áp dụng ngay sysctl -p
03 / PROTOCOL

Thay đổi giao thức — thoát khỏi SOCKS5 TCP "trần"

SOCKS5 thuần TCP dễ bị ISP Việt Nam throttle qua DPI. Chuyển sang giao thức có UDP hoặc obfuscation:

WireGuard
✦ TỐT NHẤT
UDP-based, latency cực thấp, lý tưởng cho TikTok streaming
Shadowsocks / V2Ray
✓ TỐT
Mã hóa, vượt DPI mượt mà, VMess/VLESS ổn định
SOCKS5 TCP
— CƠ BẢN
Dễ bị throttle, không có obfuscation, phụ thuộc ISP

05 / SYSCTL & MTU

Tối ưu Kernel Linux & MTU

Điều chỉnh kernel và MTU để tránh phân mảnh gói tin, tăng buffer cho kết nối đường dài.

# Tăng buffer TCP net.core.rmem_max = 134217728 net.core.wmem_max = 134217728 net.ipv4.tcp_rmem = 4096 87380 67108864 # MTU tối ưu (chọn 1) ip link set eth0 mtu 1400 # hoặc 1450
