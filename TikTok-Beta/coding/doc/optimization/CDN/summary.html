Trước & Sau khi có CDN

❌ TRƯỚC CDN (trước ~1998)
👤 User VN ————————————→ 🖥️ Server Mỹ ~300ms
Gói tin phải vượt Thái Bình Dương
👤 User EU ————————→ 🖥️ Server Mỹ ~150ms
  • 🔴 Một server gốc duy nhất cho toàn thế giới
  • 🔴 Độ trễ cao với người dùng ở xa (200–400ms)
  • 🔴 Traffic tăng đột biến → server crash (Slashdot Effect)
  • 🔴 Single Point of Failure — server chết = toàn bộ sập
  • 🔴 Video streaming gần như không khả thi ở quy mô lớn
  • 🔴 World Cup 1998 & bầu cử Mỹ làm quá tải Internet toàn cầu
✅ SAU CDN (1998 – nay)
👤 User VN → ⚡ Edge SG ~20ms
Chỉ cần đến Singapore hoặc HN/HCM
👤 User EU → ⚡ Edge FRA ~8ms
  • 🟢 Hàng nghìn edge servers trên toàn thế giới
  • 🟢 Độ trễ thấp bất kể người dùng ở đâu (dưới 50ms)
  • 🟢 Super Bowl, World Cup không làm sập hệ thống nữa
  • 🟢 PoP chết → tự failover sang PoP kế bên
  • 🟢 Netflix 4K, YouTube, TikTok — streaming toàn cầu
  • 🟢 Băng thông Origin giảm 60–80% nhờ caching

// 03 — Khái niệm

Các khái niệm cốt lõi

🖥️
ORIGIN SERVER

Máy chủ gốc

Nơi lưu trữ phiên bản gốc và chính xác nhất của nội dung. Là nguồn sự thật duy nhất (source of truth). Mọi thay đổi thực hiện trên đây trước khi CDN đồng bộ.

⚡
EDGE SERVER / PoP

Máy chủ biên

Các server đặt tại nhiều thành phố lớn, IXP (Internet Exchange Point). Lưu cache nội dung và phục vụ trực tiếp người dùng gần đó — không cần hỏi Origin.

💾
CACHING

Lưu bộ đệm

Bản sao tạm thời của nội dung tại Edge. Khi người dùng request, Edge kiểm tra cache trước. Cache Hit → trả ngay. Cache Miss → lấy từ Origin → lưu lại → trả.

⏱️
TTL — TIME TO LIVE

Thời gian sống

Khoảng thời gian Edge giữ bản cache trước khi xóa và lấy lại từ Origin. TTL ngắn = nội dung luôn mới. TTL dài = ít traffic về Origin nhưng có thể stale.

🔀
ANYCAST ROUTING

Định tuyến Anycast

Nhiều PoP dùng chung một địa chỉ IP. Khi request đến, router Internet tự động chọn PoP gần nhất về mặt mạng. Người dùng không cần biết họ đang kết nối đến đâu.

📡
BGP

Border Gateway Protocol

Giao thức định tuyến của Internet. CDN dùng BGP để "quảng bá" rằng mình có thể phục vụ nội dung từ nhiều điểm khác nhau trên thế giới.

🔄
PULL vs PUSH

Hai chế độ CDN

Pull: CDN tự động kéo nội dung về khi có request đầu tiên (phổ biến). Push: Bạn chủ động đẩy file lên CDN trước khi có request (cho file lớn, ít thay đổi).

🧹
PURGE / INVALIDATION

Xóa cache

Khi nội dung cập nhật, bạn cần "purge" cache tại Edge để buộc CDN lấy bản mới từ Origin. Có thể xóa theo URL cụ thể, wildcard, hoặc toàn bộ.

// 04 — Luồng xử lý

Cache Hit vs Cache Miss

✅ CACHE HIT — Nhanh
👤
User gửi request REQUEST
⚡
Edge Server nhận & kiểm tra cache
💾
Cache tìm thấy bản sao (còn TTL) HIT
🚀
Trả response trực tiếp cho User <20ms

Origin server không bị đụng đến. Tiết kiệm băng thông và tài nguyên.

⚠️ CACHE MISS — Chậm hơn
👤
User gửi request REQUEST
⚡
Edge Server kiểm tra cache
❌
Không có cache (miss / expired) MISS
🖥️
Lấy từ Origin Server ~200ms
💾
Lưu vào cache + trả cho User CACHED

Lần sau request tương tự sẽ là Cache Hit — chỉ lần đầu mới chậm.

// 06 — Ứng dụng

Use Cases phổ biến

🎬

Video Streaming

Netflix, YouTube, TikTok, Twitch. HLS/DASH adaptive bitrate. Live event hàng triệu người xem đồng thời.

🛒

E-commerce

Tải ảnh sản phẩm cực nhanh (Shopee, Amazon). Tăng tỉ lệ chuyển đổi. 100ms delay = 1% revenue giảm.

🌐

Static Assets

HTML, CSS, JS, fonts, images. JAMstack (Vercel, Netlify). Blog, landing page, SPA.

🎮

Game & Software

Steam, Epic Games. Update game hàng GB đến triệu người. Asset 3D, patch realtime.

🛡️

DDoS Protection

CDN hấp thụ tấn công Tbps. WAF lọc bot, brute force. Cloudflare, Akamai Shield.

⚙️

API Acceleration

Cache JSON response tại Edge. Giảm load backend. Route optimization qua CDN core network.

🧩

Edge Computing

Cloudflare Workers, Lambda@Edge. Chạy code ngay tại PoP. Sub-millisecond logic execution.

📱

Mobile & IoT

App toàn cầu phục vụ từ PoP gần nhất. Tiết kiệm data người dùng. Giảm latency API calls.

// 07 — Triết lý thiết kế

Design Philosophy

📍
"Bring content closer to users"

Triết lý cốt lõi: đưa dữ liệu đến gần người dùng thay vì bắt người dùng đến gần dữ liệu. Ánh sáng qua cáp quang có giới hạn vật lý ~200,000 km/s. CDN cắt ngắn hành trình đó.

🌐
"Distributed over Centralized"

Hệ thống phân tán luôn scalable hơn tập trung. Khi traffic tăng, thêm PoP thay vì nâng cấp một server duy nhất. Không có Single Point of Failure.

💾
"Cache everything possible"

Cache là vũ khí mạnh nhất của CDN. Càng nhiều người xem cùng nội dung → cache hit rate càng cao → hệ thống càng nhanh và rẻ. Network effect của caching.

🔧
"Fail gracefully"

Lỗi một node không được ảnh hưởng toàn hệ thống. PoP down → tự failover sang PoP kế bên. Tính sẵn sàng cao (high availability) là không thể thương lượng.
