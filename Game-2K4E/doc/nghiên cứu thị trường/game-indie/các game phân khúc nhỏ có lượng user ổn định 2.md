# Mô hình thu hút user từ các tựa game trên

Dưới góc nhìn **market research + growth model**, các game indie bạn thấy ở trên thực ra không “random thành công” — chúng follow những **mô hình thu hút user (user acquisition loops)** khá rõ ràng.

Mình sẽ bóc tách theo **cách user đến với game**, không phải chỉ gameplay.

---

# 🧠 1. Mô hình 1: **Viral Loop (Self-propagating growth)**

### Case:

- **Lethal Company**
- **Content Warning**
- **Buckshot Roulette**

---

## 🔁 Cơ chế:

```
1 user chơi → tạo tình huống hài/horror → clip → đăng TikTok/YouTube
→ người khác thấy → mua game → tạo clip tiếp
```

👉 Đây là **growth loop tự nhân bản**, gần như không cần ads.

---

## ⚙️ Thiết kế đứng sau:

- Gameplay phải:
  - **không đoán trước (emergent)**
  - tạo moment “WTF / funny / scream”

- Không cần đồ họa đẹp → chỉ cần **reaction mạnh**

---

## 🎯 Insight:

- Không phải “game hay” → mà là **game tạo content được**
- Dev đang outsource marketing cho player

---

## 📊 KPI quan trọng:

- Clip frequency / hour gameplay
- Share rate
- Watch → install conversion

---

# 🧠 2. Mô hình 2: **Addiction Loop (Retention-driven growth)**

### Case:

- **Balatro**
- **Backpack Battles**

---

## 🔁 Cơ chế:

```
User thử game → bị cuốn vào loop → chơi lâu → review tốt
→ Steam recommend → organic traffic tăng
```

👉 Không viral mạnh, nhưng **scale bền vững**

---

## ⚙️ Thiết kế:

- Core loop:
  - ngắn (5–20 phút/run)
  - có “progress nhỏ liên tục”

- RNG + build diversity → replay vô hạn

---

## 🎯 Insight:

- Growth đến từ:
  - **Steam algorithm (review + playtime)**

- Đây là kiểu:

  > “slow burn nhưng ăn dài”

---

## 📊 KPI:

- Avg session length
- D1 / D7 retention
- Review score (>90% cực quan trọng)

---

# 🧠 3. Mô hình 3: **Social Pull (network-driven acquisition)**

### Case:

- **Content Warning**
- **Lethal Company**

---

## 🔁 Cơ chế:

```
1 người mua → rủ 3–4 bạn → cả nhóm cùng mua
```

👉 Đây là **K-factor > 1 (growth exponential)**

---

## ⚙️ Thiết kế:

- Bắt buộc:
  - Co-op (2–6 người)
  - Fun hơn khi đông

- Tạo cảm giác:
  - “chơi 1 mình là mất 80% fun”

---

## 🎯 Insight:

- User acquisition = **friend pressure**
- Đây là dạng:

  > “marketing miễn phí qua bạn bè”

---

## 📊 KPI:

- Avg players per session
- Invite rate
- Conversion per group

---

# 🧠 4. Mô hình 4: **Hybrid Hook (Concept-driven acquisition)**

### Case:

- **Palworld**
- **Deep Rock Galactic: Survivor**

---

## 🔁 Cơ chế:

```
User thấy concept → tò mò → thử game
```

VD:

- “Pokémon + súng”
- “Vampire Survivors + mining shooter”

---

## ⚙️ Thiết kế:

- Hook phải:
  - dễ hiểu trong 1 câu
  - gây tranh cãi / tò mò

---

## 🎯 Insight:

- Không cần gameplay quá mới
- Chỉ cần:

  > “mix đúng thứ người ta đã thích”

---

## 📊 KPI:

- CTR trailer
- Wishlist rate
- Conversion day-1

---

# 🧠 5. Mô hình 5: **Meme / Novelty (attention hack)**

### Case:

- **Banana**

---

## 🔁 Cơ chế:

```
Meme → tò mò → tải → lan truyền → spike user
```

---

## ⚙️ Thiết kế:

- Gameplay gần như không quan trọng
- Quan trọng:
  - weird / absurd
  - dễ lan truyền

---

## 🎯 Insight:

- Đây là:

  > “attention economy game”

---

## 📊 KPI:

- Impressions
- Viral coefficient
- (Retention thường thấp)

# 🚀 Nếu bạn làm game (rất thực tế)

Bạn nên chọn **1 model ngay từ đầu**, không phải làm xong rồi mới nghĩ marketing.

Ví dụ:

- Muốn viral → thiết kế game cho streamer
- Muốn bền → tối ưu loop + retention
- Muốn scale nhanh → làm co-op

# Mô hình kiếm tiền từ các tựa game trên

Tiếp theo lớp **growth (thu hút user)** thì lớp quan trọng không kém là **monetization (kiếm tiền)** — và với game phân khúc nhỏ, cách kiếm tiền thường rất “khôn” chứ không brute force như AAA.

Mình sẽ bóc tách theo **mô hình kiếm tiền thực tế đang hoạt động**, gắn trực tiếp với các game bạn đã thấy.

---

# 💰 1. Mô hình: **Premium đơn giản (Pay once, play forever)**

### Case:

- **Balatro**
- **Buckshot Roulette**
- **Backpack Battles**

---

## ⚙️ Cách kiếm tiền:

```id="kpm0i8"
User trả 1 lần (5$–15$) → chơi full game
```

---

## 🧠 Tại sao nó vẫn sống tốt?

- Giá rẻ → **friction thấp**
- Loop gây nghiện → **value/hour cực cao**
- Review tốt → kéo thêm user mới

---

## 📊 Công thức doanh thu:

```id="3qv2l8"
Revenue = Price × Volume (sales)
```

---

## 🎯 Insight:

- Indie thắng bằng:

  > “giá rẻ + addictive → volume lớn”

- Không cần whale như mobile game

---

## ⚠️ Rủi ro:

- Không có recurring revenue
- Phụ thuộc launch + Steam algo

---

# 💰 2. Mô hình: **Viral premium (low price × massive volume)**

### Case:

- **Lethal Company**
- **Content Warning**

---

## ⚙️ Cách kiếm tiền:

```id="3h1z2v"
Giá thấp (~5$–10$) + viral → bán cực nhiều bản
```

---

## 🧠 Điểm đặc biệt:

- Không maximize ARPU
- Mà maximize:

  > **total player count**

---

## 🎯 Insight:

- Vì là co-op:
  - 1 người mua → kéo 3–4 người mua

- Giá thấp giúp:
  - giảm friction khi rủ bạn

---

## 📊 Thực tế:

- Đây là model:

  > “network effect monetization”

---

# 💰 3. Mô hình: **Early Access + bán dần (cashflow sớm)**


## 🧠 Lợi ích:
